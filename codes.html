Примеры кода в консоли

(config-if): mdix auto
включить автоматическое перекрещивание Ethernet кабеля
(config-if): duplex {type}
переключение типа дуплекса
(config-if): speed {type}
изменение скорости передачи данных на интерфейсе

Базовая настройка оборудования

>: enable
перейти в привилегированный режим EXEC
>: ping {ip_address}
пингануть ip_address
>: traceroute {ip_address}
посмотреть как пакет идет по сети при пинге {ip_address}
#: configure terminal
перейти в режим глобальной конфигурации
#: show running-config
посмотреть файл текущей конфигурации
#: show startup-config
посмотреть файл загрузочной конфигурации
#: show ip interface brief
показать настройку ipv4 всех интерфейсов на оборудовании
#: show ipv6 interface brief
показать настройку ipv6 всех интерфейсов на оборудовании
#: show running-config interface {interface_name}
просмотр настроек интерфейса
#: show history
показать историю вводимых команд
#: copy {from_file} {to_file}
копирование содержимого from_file в to_file
#: erase startup-config
очистить файл загрузочной конфигурации
#: reload
перезагрузка устройства
(config): hostname {hostname}
изменить имя устройства
(config): no hostname {hostname}
удалить имя устройства
(config): line console 0
перейти в режим настройки командной строки
(config): line vty {from_index} {to_index}
перейти в режим настройки линий VTY (виртуальных терминальных линий)
(config): interface {interface_name}
перейти в режим настройки интерфейса
(config): service password-encryption
включить шифрование паролей в файле текущей конфигурации
(config): enable secret {password}
задать пароль для привилегированного режима EXEC
(config): no ip domain-lookup
отключение разрешения доменного имени
(config-line): password {password}
задать пароль для пользовательского режима EXEC
(config-line): login
применить пароль (обязательно перед этим задать какой-то пароль)
(config-line): logging sunchronous
синхронизация логирования
(config-if): ip address {ip_address} {subnet_mask}
настройка IP-адреса на интерфейсе
(config-if): no shutdown
включение интерфейса (по умолчанию все выключены)
(любой режим): exit
перейти в предыдущий по иерархии режим
(любой режим): end
перейти в привилегированный режим EXEC

Введение в маршрутизацию

#: show ip route
просмотр таблицы ipv4 маршрутизации
#: show ipv6 route
просмотр таблицы ipv6 маршрутизации
(config): ip route {subnet} {subnet_mask} {ip-address | exit-intf} [distance]
создание статического ipv4 маршрута для сети subnet с административным расстоянием distance
(config): ipv6 route {subnet}/{prefix-length} {ip-address | exit-intf} [distance]
создание статического ipv6 маршрута для сети subnet с административным расстоянием distance
(config): interface loopback {number}
создать loopback интерфейс
(config-if): ipv6 address {ipv6_address}/{prefix_length}
настройка ipv6 адреса на интерфейсе
(config-if): description {description}
задать описание для интерфейса

Маршрутизация

#: show ip protocols
просмотр настроек протоколов маршрутизации
(config): router rip
перейти в режим настройки RIP
(config): passive-interface default
сделать все интерфейсы пассивными
(config-if): no passive-interface
сделать интерфейс активным (по умолчанию)
(config-router): version 2
перейти к RIPv2 (по умолчанию используется RIP первой версии)
(config-router): network {network}
указать, о какой сети рассказывать другим маршрутизаторам через RIP
(config-router): no auto-summary
отключить объединение сетей в классовые суперсети в таблице маршрутизации
(config-router): passive-interface {interface_name}
сделать интерфейс пассивным
(config-router): default-information originate
включить маршрут по умолчанию в распространяемую RIP информацию

IP-адресация

(config): ipv6 unicast-routing
включение маршрутизации IPv6
(config-if): ipv6 address {ipv6_address} link-local
настройка link-local ipv6 адреса на интерфейсе

DHCP

#: show ip dhcp binding
посмотреть привязки выдданых Ipv4 адресов клиентам с их MAC-адресами
#: show ip dhcp server statistics
посмотреть статистику работы dhcp сервера
#: show ipv6 dhcp pool
посмотреть настройки пулов ipv6 адресов
#: show ipv6 shcp binding
посмотреть привязки ipv6 адресов клиентам
(config): ip dhcp excluded-address {from_ip_address} {to_ip_address}
исключить диапазон ipv4 адресов из раздачи
(config): ip dhcp pool {pool_name}
создать dhcpv4 пул
(config): ipv6 dhcp pool {pool_name}
создать dhcpv6 пул
(config): no service dhcp
отключение dhcp сервера
(config): service dhcp
включение dhcp сервера (по умолчанию)
(config-if): ip helper-address {ip_address}
настройка ретрансляции на интерфейсе (для DNS и DHCP серверов в других сетях)
(config-if): ipv6 dhcp relay destination {ipv6_address} {exit_intf}
настройка ipv6 ретрансляции на интерфейсе
(config-if): ip address dhcp
настройка интерфейса в качестве DHCP-клиента (для получения IPv4 адреса через DHCP)
(config-if): ipv6 nd other-config-flag
установка флага O в Router Advertisement сообщении маршутизатора (для работы в SLAAC + DHCPv6 режиме) (он же Stateless DHCPv6)
(config-if): ipv6 nd managed-config-flag
установка флага M в Router Advertisement сообщении маршутизатора (для работы в DHCPv6 режиме) (он же Stateful DHCPv6)
(config-if): ipv6 nd prefix default no-autoconfig
убрать флаг A в Router Advertisement сообщении маршутизатора (для работы в DHCPv6 режиме) (он же Stateful DHCPv6)
(config-if): ipv6 dhcp server {pool_name}
привязать ipv6 пул к интерфейсу
(config-if): ipv6 enable
сгенерировать link-local ipv6 адрес на интерфейсе
(config-if): ipv6 address autoconfig
получить ipv6 адрес через SLAAC
(config-if): dhcp ipv6 address
настроить получение ipv6 адреса через Stateful DHCPv6
(dhcp-config): network {network} {network_mask}
задать пул ipv4 адресов для выдачи
(dhcp-config): default-router {ip_address}
задать шлюз по умолчанию
(dhcp-config): domain-name {domain_name}
задать доменное имя
(dhcp-config): lease {days [hours [minutes]] | infinite}
задать время аренды IP-адреса
(config-dhcpv6): dns-server {ipv6-address}
настройка dns сервера
(config-dhcpv6): domain-name {domain_name}
задать доменное имя
(config-dhcpv6): address prefix {prefix}/{prefix_length}
задать префикс для выдачи ipv6 адресов

VLAN

#: delete flash:vlan.dat
удалить все настроенные vlan с 1 по 1001
#: show vlan brief
просмотр настроек vlan и принадлежности портов к vlan
#: show vlan summary
просмотр статистики по vlan
#: show vlan name {vlan_name}
просмотр информации о конкретном vlan
#: show interfaces {interface_name} switchport
просмотр информации о настройке vlan на интерфейсе
#: show interfaces trunk
показать список и настройки trunk портов
(config): vlan {id}
создание vlan
(config): no vlan {id}
удаление vlan
(config-if): switchport mode access
перевести vlan в режим для приема данных
(config-if): switchport access vlan {id}
ассоциировать порт с vlan с заданным id
(config-if): switchport voice vlan {id}
ассоциировать порт с голосовым vlan с заданным id
(config-if): no switchport access vlan
вернуть порт в vlan по умолчанию (в vlan 1)
(config-if): switchport mode trunk
перевести порт в состояние trunk порта
(config-if): switchport trunk encapsulation dot1q
включить информацию об id vlan в кадр Ethernet
(config-if): switchport trunk native vlan {id}
ассоциировать порт с native vlan
(config-if): no switchport trunk native vlan
сброс native vlan на порту
(config-if): switchport trunk allowed vlan {vlan1, vlan2, …}
задать список допустимых vlan на этом порту
(config-if): no switchport trunk allowed vlan
сброс настроек по допустимым vlan на этом порту
(config-vlan): name {name}
задать имя для vlan
